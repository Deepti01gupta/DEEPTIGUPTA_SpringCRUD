REQUEST/RESPONSE FLOW - EXAMPLE: POST /students
================================================================================

CLIENT (Postman):
┌─────────────────────────────────────────────┐
│ POST http://localhost:8080/api/students     │
│ Content-Type: application/json              │
│                                             │
│ Body: {                                     │
│   "name": "New Student",                    │
│   "email": "new@example.com",               │
│   "course": "Data Science"                  │
│ }                                           │
└────────────────────┬────────────────────────┘
                     │
         ┌───────────▼───────────┐
         │ REQUEST VALIDATION    │
         │ - Parse JSON          │
         │ - Verify format       │
         └───────────┬───────────┘
                     │
         ┌───────────▼────────────────┐
         │ StudentController          │
         │ - @PostMapping             │
         │ - Extract body params      │
         │ - Call service layer       │
         └───────────┬────────────────┘
                     │
         ┌───────────▼──────────────────┐
         │ StudentService               │
         │ - Validate input fields      │
         │ - Check for null/empty       │
         │ - Call repository.save()     │
         └───────────┬──────────────────┘
                     │
         ┌───────────▼────────────────────────┐
         │ StudentRepository                  │
         │ - Build SQL: INSERT INTO student   │
         │ - Execute via JdbcTemplate         │
         │ - Bind parameters                  │
         └───────────┬────────────────────────┘
                     │
         ┌───────────▼───────────────────┐
         │ PostgreSQL Database           │
         │ - Execute INSERT              │
         │ - Generate auto-increment ID  │
         │ - Return affected rows        │
         └───────────┬───────────────────┘
                     │
         ┌───────────▼────────────────┐
         │ Response Object            │
         │ - Student with ID = 6      │
         └───────────┬────────────────┘
                     │

================================================================================
HTTP STATUS CODES USED
================================================================================

200 OK
  - GET /students
  - GET /students/{id} (found)
  - PUT /students/{id} (successful update)

201 CREATED
  - POST /students (student created)

204 NO CONTENT
  - DELETE /students/{id} (successful deletion)

400 BAD REQUEST
  - Invalid request format
  - Missing required fields

404 NOT FOUND
  - GET /students/{id} (not found)
  - PUT /students/{id} (not found)
  - DELETE /students/{id} (not found)

500 INTERNAL SERVER ERROR
  - Database connection errors
  - Validation errors
  - Unexpected server errors

DEPLOYMENT WORKFLOW
================================================================================

1. BUILD PHASE:
   mvn clean install
   └─ Compiles code
   └─ Runs tests
   └─ Creates JAR file (target/backend-0.0.1-SNAPSHOT.jar)

2. RUN PHASE:
   mvn spring-boot:run
   OR
   java -jar target/backend-0.0.1-SNAPSHOT.jar
   └─ Starts embedded Tomcat on port 8080
   └─ Initializes connection pool
   └─ Loads context path /api

3. TEST PHASE:
   Use Postman to test endpoints
   └─ Verify all 5 endpoints
   └─ Check status codes
   └─ Validate responses

4. DATABASE VERIFICATION:
   psql -U postgres -d mydb
   SELECT * FROM student;
